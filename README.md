# Pookie — AI PM Interview Practice

A conversational, time-boxed AI interviewer for practicing AI PM interviews. Focuses on AI system design questions with rubric-based evaluation grounded in curated references.

## Features (v0)

- **Fresh questions** generated by LLM each session (no memorised teardowns)
- **Pookie** — an interviewer persona that probes, nudges, and follows up
- **45-minute timer** with countdown
- **Socratic hints** — 3 per session, probe thinking without giving answers
- **End-of-session evaluation** — DASME rubric scores, strengths, gaps, model answer
- **Session dashboard** — history and score trends

## Stack

- Next.js 14 (App Router + Server Actions)
- Supabase (Postgres + Auth + RLS)
- Anthropic Claude Sonnet
- Tailwind CSS + shadcn/ui
- Zod validation

## Setup

### 1. Clone and install

```bash
cd pookie
npm install
```

### 2. Create a Supabase project

1. Go to [supabase.com](https://supabase.com) and create a free project
2. In the SQL editor, run the entire contents of `lib/supabase/schema.sql`
3. Copy your project URL and keys from **Settings → API**

### 3. Configure environment variables

```bash
cp .env.local.example .env.local
```

Fill in `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
ANTHROPIC_API_KEY=your-anthropic-api-key
```

Get your Anthropic API key at [console.anthropic.com](https://console.anthropic.com).

### 4. Run locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Usage

1. Open the app → sign in with your email (magic link)
2. Select question type, role level, and timer
3. Click **Start Interview** — Pookie generates a fresh question
4. Type your response in the chat box
5. Use **Hint** button if stuck (3 per session, Socratic only)
6. Click **End & Evaluate** when done
7. Review your DASME rubric scores, strengths, gaps, and model answer

## Configuration

Edit `config/llm.ts` to swap models or change defaults without touching other code:

```ts
export const llmConfig: LLMConfig = {
  provider: 'anthropic',
  model: 'claude-sonnet-4-5',   // swap model here
  maxTokens: 1500,
  temperature: 0.7,
}

export const defaultInterviewConfig: InterviewConfig = {
  timerMinutes: 45,             // default timer
  questionType: 'ai_system_design',
  role: 'sr_pm',
  maxHints: 3,
}
```

## Content Files

All rubrics, frameworks, and sample answers live in `content/`. Edit them directly — no code changes needed.

```
content/
├── rubrics/
│   └── ai-system-design.md      ← DASME scoring rubric (1-5 per dimension)
├── frameworks/
│   └── ai-system-design.md      ← DASME framework used for hints and Q-gen
└── sample-answers/
    └── ai-system-design.md      ← Reference answer for evaluation grounding
```

---

## Important Notes

- **Rubric coverage ≠ real interview pass rate.** This tool measures framework coverage, not the unpredictability of real loops.
- **Do not commit `.env.local`** — it is already in `.gitignore`.
- LLM evaluation is grounded in curated `.md` files to reduce hallucination risk, but always verify technical trade-offs independently.
